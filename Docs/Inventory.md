# 인벤토리 시스템 설계

## 개요

아이템과 장비를 관리하는 시스템. 캐릭터와 아이템 모두 **아이템 슬롯**을 보유할 수 있으며, 슬롯은 재귀적으로 중첩되는 구조를 가진다.

---

## 아이템 슬롯 (ItemSlot)

캐릭터 또는 아이템이 보유하는 저장 단위.

### 속성

| 속성 | 설명 |
|------|------|
| 허용 아이템 종류 | 이 슬롯에 보관 가능한 아이템 종류 목록. 비어 있으면 모든 종류 허용 |
| 최대 무게 | 이 슬롯이 수용할 수 있는 최대 무게 |
| 보관 중인 아이템 | 현재 슬롯에 들어 있는 아이템 목록 |
| 슬롯 수 | 보관 가능한 아이템 스택의 최대 칸 수 |

### 중첩 구조

```
캐릭터
 └─ ItemSlot (인벤토리)
     └─ Item (탄약상자)
         └─ ItemSlot (탄약상자 내부, 총알 종류만 허용)
             └─ Item (9mm 총알 x30)
```

기술적으로 임의 깊이까지 중첩이 가능하지만, 실제로는 다음 두 가지 메커니즘으로 통제한다.

- **허용 아이템 종류 제한**: 슬롯이 받을 수 있는 아이템 종류를 명시적으로 제한
- **무게 시스템**: 아이템을 보관할수록 상위 슬롯의 무게가 증가하여 과도한 중첩을 자연스럽게 억제

---

## 아이템 (Item)

### 고정 데이터 (Static / ScriptableObject 등)

| 필드 | 설명 |
|------|------|
| 이름 | 아이템 표시명 |
| 아이콘 | UI에 표시할 스프라이트 |
| 종류 (ItemType) | 아이템 분류 (예: 총기, 탄약, 소모품, 장비 등) |
| 최대 중첩 수 | 한 스택에 쌓을 수 있는 최대 개수 (1이면 중첩 불가) |
| 아이템 슬롯 보유 정보 | 이 아이템이 내부에 가지는 슬롯 정의 목록 (없을 수도 있음) |
| 기본 무게 | 아이템 자체의 기본 무게 (내용물 제외) |
| 효과 | 사용/장착 시 적용되는 효과 정의 목록 |

### 동적 데이터 (Runtime)

| 필드 | 설명 |
|------|------|
| 현재 중첩 수 | 현재 스택에 쌓인 개수 |
| 슬롯 내 아이템 | 이 아이템이 보유한 각 ItemSlot의 현재 상태 |
| 내부 합산 무게 | 내부 슬롯에 보관된 아이템 무게의 합산값 (기본 무게에 더해져 최종 무게를 구성) |

> **최종 무게** = 기본 무게 + 내부 합산 무게

## 캐릭터의 슬롯 구성

캐릭터는 두 종류의 아이템 슬롯을 가진다.

### 범용 인벤토리 슬롯

- 종류 제한 없이 대부분의 아이템을 보관할 수 있는 일반 저장공간
- 총 무게 및 칸 수로 용량 제한

### 장비 슬롯

- 착용 가능한 특정 종류의 아이템만 허용 (예: 머리 슬롯에는 헬멧 종류만)
- 슬롯 수는 보통 1개로 고정
- 착용된 아이템의 효과가 캐릭터에 적용됨

### 스탯

캐릭터는 스탯 시스템을 가진다. 스탯의 상세 설계는 별도 문서에서 다룬다.

---

## 아이템 종류 (ItemType) 예시

> 실제 종류는 게임 콘텐츠에 따라 확장

- `Firearm` — 총기류
- `Ammo` — 탄약류
- `Consumable` — 소모품 (회복 아이템 등)
- `Equipment` — 장비류 (방어구, 악세서리 등)
- `Container` — 내부 슬롯을 가지는 컨테이너형 아이템 (탄약상자, 가방 등)
- `Misc` — 기타

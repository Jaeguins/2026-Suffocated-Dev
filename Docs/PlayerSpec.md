# PlayerSpec — 캐릭터 스탯 시스템 설계

## 개요

캐릭터(플레이어, 적 공통)의 스탯을 관리하는 시스템.  
각 스탯은 **기본값(Base)** 하나와, 여러 개의 **변동자(Modifier)** 로 구성된다.

---

## 스탯 구조

### 기본값 (Base)

- 초기 설정 시 1회 결정되며 이후 불변.
- `float` 타입.

### 변동자 (Modifier)

변동자는 **원인(Source)** 과 **변동량(Delta)** 의 쌍으로 구성된다.

#### 원인 (Source)

| 필드 | 타입 | 설명 |
|------|------|------|
| `type` | enum / string | 원인의 종류 (예: `Equipment`, `Buff`, `Passive`, `StatusEffect`) |
| `id`   | string / int  | 같은 종류 내 개별 식별자 (예: 장비 아이템 ID, 버프 ID) |

동일 `type` + 다른 `id` 는 별개의 변동자로 공존 가능.  
동일 `type` + 동일 `id` 는 덮어쓰기(갱신)로 처리.

#### 변동량 (Delta)

| 연산 종류 | 설명 |
|-----------|------|
| **합연산 (Additive)** | 기본값에 더해지는 flat 수치 |
| **곱연산 (Multiplicative)** | 배율로 적용, 1.0 기준 (예: +20% → 0.2, ×1.5 → 1.5) |

---

## 최종값 계산 공식

```
최종값 = (Base + Σ additive) × (1 + Σ multiplicative_delta)
```

> 곱연산 값은 `1` 기준의 **증감분**으로 저장한다.  
> 예: 20% 보너스 → `0.2`, 50% 보너스 → `0.5`  
> 합산 후 `1 +` 하여 최종 배율로 변환.

### 계산 예시

| 항목 | 값 |
|------|----|
| 기본 체력 (Base) | 100 |
| 합연산 A | +12 |
| 합연산 B | +13 |
| 곱연산 A | +20% (0.2) |
| 곱연산 B | +50% (0.5) |

```
(100 + 12 + 13) × (1 + 0.2 + 0.5)
= 125 × 1.7
= 212.5
```

---

## 캐싱

- 변동자 목록이 변경되지 않으면 최종값을 캐시하여 재사용.
- 변동자 추가 / 제거 / 갱신 시 캐시 무효화(dirty flag).
- `Get()` 호출 시 dirty이면 재계산 후 캐시 저장, 그렇지 않으면 캐시값 반환.

## 스탯 목록

스탯은 플레이어와 적이 공유하는 구조이며, 보유 여부는 각 엔티티 설정에 따른다.

### 공격 관련

| 스탯 키 | 설명 | 단위 |
|---------|------|------|
| `WeaponAttack` | 무기 기본 공격력 | flat |
| `MeleeMultiplier` | 근접 공격 배율 | 배율 |
| `RangedMultiplier` | 원거리 공격 배율 | 배율 |

### 속성 관련

| 스탯 키 | 설명 | 단위 |
|---------|------|------|
| `FireMultiplier` | 화염 속성 공격 배율 | 배율 |
| `IceMultiplier` | 냉기 속성 공격 배율 | 배율 |
| *(이하 속성별 확장)* | | |

### 대상 유형 관련

| 스탯 키 | 설명 | 단위 |
|---------|------|------|
| `BossMultiplier` | 보스 대상 공격 배율 | 배율 |
| `EliteMultiplier` | 엘리트 대상 공격 배율 | 배율 |

### 방어 관련

| 스탯 키 | 설명 | 단위 |
|---------|------|------|
| `Defense` | 방어력 (데미지 감소) | flat 또는 배율, 계산식 별도 |
| `FireResistance` | 화염 내성 | 배율 (감소분) |
| `IceResistance` | 냉기 내성 | 배율 (감소분) |
| *(이하 속성별 확장)* | | |

### 생존 관련

| 스탯 키 | 설명 | 단위 |
|---------|------|------|
| `MaxHp` | 최대 체력 | flat |
| `MoveSpeed` | 이동 속도 | flat |

## 실질 데미지 계산 예시

> 플레이어가 **보스**인 적에게 **화염 근접공격**을 하는 경우

```
실질 데미지
  = 플레이어.WeaponAttack (최종값)
  × 플레이어.MeleeMultiplier (최종값)
  × 플레이어.BossMultiplier (최종값)
  × 플레이어.FireMultiplier (최종값)
  ÷ 적.FireResistance (최종값)
  ÷ 적.Defense (최종값)
```

- 각 항목은 모두 **별개의 Spec 스탯**으로, 플레이어와 적 각각이 독립적으로 보유.
- 나눗셈 항목(`FireResistance`, `Defense`)은 곱연산 시 역수로 변환하거나, 계산식에서 명시적으로 나누기 처리.

---

## 구현 메모

- `SpecStat` 클래스: 단일 스탯 (base + modifiers + cache)
- `SpecSheet` 클래스 또는 Dictionary: 엔티티가 보유한 스탯 전체 집합
- Source 동일성 비교는 `(type, id)` 쌍으로 판단
- 수치 타입은 전부 `float` (오차 허용)
- 스탯 키는 enum 또는 string constant로 중앙 관리 권장

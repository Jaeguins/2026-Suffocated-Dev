- [x] 로딩 씬 집결 공간: 로비 씬 한쪽에 로딩씬 소속 지형 배치, 미션 진입 조건 충족 시 하이라이팅

- [x] 로딩 UI: 각 플레이어별 로딩바 + 별도 팁 텍스트
- [ ] **NGO Scene Management 채용 여부** ← 의사결정 필요
  - **채용 시 장점**
    - 호스트의 `LoadScene` / `UnloadScene` 호출이 모든 클라이언트에 자동 동기화
    - `NetworkSceneManager`가 각 클라이언트의 씬 로드 상태를 자동 추적
    - `OnLoadComplete` 이벤트로 전원 로드 완료 감지 가능
  - **채용 시 단점**
    - 씬 로드/언로드 권한이 서버(호스트)에만 있어 클라이언트 독립 씬 조작 불가
    - Additive 씬 전환 + 5초 카운트 + 이동 블록 등 커스텀 절차를 NGO 위에 별도 레이어로 구현해야 함
  - **미채용 시**
    - 씬 로드를 모든 클라이언트에 RPC로 수동 전달해야 함
    - 커스텀 로딩 로직 구현 자유도가 높음

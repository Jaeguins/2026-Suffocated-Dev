# 게임 개요 (Summary)

> 최종 수정: 2026-03-30

---

## 장르 및 컨셉

- **장르:** 4인 협동 익스트랙션 슈터
- **시점:** 2D 탑뷰
- **아트 스타일:** 픽셀아트 (기본 픽셀 사이즈 미결, 회전 등에서 픽셀 퍼펙트가 틀어질 수 있음을 인지)
- **엔진:** Unity 6000.3.11f1 (URP 2D)
- **플랫폼:** PC / Steam (주력)

---

## 네트워크 구조

- **라이브러리:** Unity Netcode for GameObjects (NGO)
- **세션 구성:** 최대 4인 협동
- **세션 소유:** 로비 진입 시 개인 세션이 자동 개설, 해당 플레이어가 호스트 역할
- **참가 방식:** 코드 입력 방식 (중계서버 없는 P2P 기반). 장기적으로 Steam SDK를 통한 친구 참여 지원 예정
- **호스트 이탈:** 마이그레이션 없음. 팝업 알림 후 모든 플레이어가 각자의 개인 로비로 이동
- **세션 잠금:** 미션 진행 중 신규 입장 불가. 로비 상태에서만 참가 가능

---

## 플로우 개요

```
메인 메뉴
 └─ 세이브 데이터 선택
     └─ 개인 세션 개설
         └─ 로비 (맵)
             ├─ 성장 관련 상호작용 대상들 (종류 미결)
             └─ 온라인 세션 관리 오브젝트 (상호작용 → UI 팝업)
                 ├─ 세션 초대 / 참가
                 └─ 미션 진입 구역 (로딩씬 소속 지형)
                     └─ 5초 카운트 → 심리스 로딩
                         └─ 플레이 씬 (미션)
                             └─ 익스트랙션 구역 (5초 카운트 → 심리스 로딩)
                                 └─ 로비 복귀
```

---

## 로비

- 별도 로비 씬으로 존재하는 인게임 맵
- 메인 메뉴에서 세이브 데이터를 선택하면 개인 세션이 열리며 로비 씬으로 진입
- **맵 크기:** 플레이어 키 2m 기준 약 200㎡. 컨텐츠 추가에 따라 확장 가능 (km 단위는 아님)
- **멀티플레이어 표시:** 같은 세션의 플레이어 캐릭터들이 동일 씬에 공존하며 보임
- **로비 내 상호작용 대상:**
  - 성장 관련 대상들 (장비, 스킬, 업그레이드 등 — 세부 종류 미결)
  - **온라인 세션 관리 오브젝트:** 상호작용 시 UI 페이지 팝업, 코드 기반 세션 초대/참가 처리

---

## 심리스 로딩 — 로비 → 미션

### 구역 구성

- 로비 씬 한쪽에 **로딩씬 소속 지형**을 배치 (물리적으로 로비 씬 내에 있으나 씬 소속은 로딩씬)
- 미션 진입 가능 조건 충족 시 해당 구역 **하이라이팅**

### 전환 절차

1. 미션 진입 조건 충족 → 구역 하이라이팅
2. 모든 플레이어가 해당 구역에 진입
3. **5초 카운트다운** (전원이 구역 내에서 유지하는 동안 카운트)
4. 카운트 완료 → 구역 탈출 블록
5. 로비 씬 언로드
6. 미션 씬 로드
7. 블록 해제 → 미션 시작

### 로딩 UI

- 각 플레이어별 **로딩바** 표시
- 별도 **팁 텍스트** 표시

---

## 심리스 로딩 — 미션 → 로비 (익스트랙션)

- 미션 맵 내 **복수의 탈출 구역** 존재
- 모든 플레이어가 탈출 구역 내에서 **5초 유지** 시 심리스 로딩으로 로비 복귀
- 전환 절차는 로비→미션 방식과 동일

---

## 세이브 시스템

- 클라이언트당 **복수 슬롯** 보유 가능
- 동시에 활성화되는 세이브는 **1인 1개**, 멀티플레이 시 최대 4명이 각각 1개씩
- 세션 진행 상태는 플레이어 각자의 세이브에 **독립적으로** 기록
- **세이브 데이터 항목:**
  - 캐릭터 스탯
  - 보유 장비
  - 창고 및 캐릭터 인벤토리 아이템
  - 통계 기록
  - 게임 진행도
  - *(추가 확장 가능)*
- **무결성:** 개발 단계에서는 외부 위변조 가능 상태 유지 (별도 검증 없음). 버전 정보에 따른 구버전 세이브 승계만 관리
- 기술 기반: `GameSaveManager` (JSON 직렬화, `Application.persistentDataPath` 저장)

---

## 미션 (플레이 씬)

### 맵 구성

- **지형:** 고정 맵
- **미션 데이터:** 절차적 생성 + 고정 데이터 혼합 (상자 내 아이템 배치 등)
- 지형 데이터와 미션 데이터는 **분리하여 관리**

### 진행 및 실패

- 진입 후 목표를 수행하고 익스트랙션(탈출)하는 구조
- **실패 조건:** 전멸
- 인원 제한 없음 (솔로 포함), **밸런싱은 추후 고려**

### 사망 처리

- 현재: 사망 상태 유지
- 관전 모드 구현 계획 있음, **구현 시점 미결**

### 적 AI

- **기본 행동:** 랜덤 배회 → 플레이어 발견 시 일정 거리 접근 후 공격
- 확장 가능한 구조로 설계 (보스 등 특수 행동 패턴 수용 고려)
- **세부 AI 구조 미결 (의사결정 필요)**

## 캐릭터

### 행동 세트

| 입력 | 행동 |
|------|------|
| Move | 이동 |
| Look | 조준 |
| Attack | 공격 |
| Interact | 상호작용 |
| Sprint | 달리기 |

> Crouch, Jump, Previous, Next는 현재 미사용

### 클래스 / 성장

- 현재 단계에서 클래스/직업 시스템 없음, 계획 확정 시 문서 확장
- 성장 구조 미결 (레벨, 장비 강화, 스킬 트리 등)

### 인벤토리

- N슬롯 구조 (슬롯 수 미결)
- 스택 가능 아이템과 불가 아이템 혼재
- 장비 시스템 및 스탯 변화 시스템 **필요하나 미결**

## 아트 / 기술

- **픽셀 사이즈:** 미결. 기본 픽셀 사이즈 표준 결정 필요. 회전 등에서 픽셀 퍼펙트가 틀어질 수 있음을 인지
- **애니메이션 툴:** 미결 (Aseprite Importer 설치됨)
- **사운드:** 미들웨어 미사용
- **카메라:** 각 로컬 플레이어 개별 추적. 컷신 등 상황에 따라 포커스 전환 가능. 분할 화면 없음
